# REFLECTION

## 1. Prompting reflection

Style A 的程式碼很少，但跑起來非常久，出來的結果也明顯無法看出任何數據趨勢，甚至連基本的標題、坐標軸名稱都沒有。另外，在只有一行的指令之下，Claude 不會主動去處理資料品質，例如即便 age 欄位有 10⁹ 量級的離群值，他依然把所有的數據的畫出來，導致大部分的數據都集中在圖片最左側，無法判讀分布。

## 2. README reflection

我認為 How to run 的部分是最難寫的。因為必須在這個專案的一開始就詳細記錄每一個步驟，如果有任何一個環節遺漏了，對於其他完全沒有做過這個專案的人來說，不論是資料的下載、環境的建立……任何細節的疏忽都可能導致讀者無法成功複製結果。

## 3. Public artifact reflection

可能會放，但是放在「課堂作業」的分類，而不會當成是個人作品集的主項目。提供這一個 repo 可以說明我能利用 venv、git、Claude Code 寫出可以重複執行的小型資料分析，不過要升級成項目，還缺以下幾件事情：

1. **統計推論**——目前只有描述統計與相關矩陣，並沒有更進一步的推論。
2. **單元測試**——`load_data.py` 沒有 pytest 驗過濾邏輯。
3. **互動視覺化**——目前是 PNG，加 Plotly 才接近現代資料分析工作流。
4. **真實研究問題**——這是公開練習資料，不是我自己主要實驗所收的資料。若我能把同一套 SOP 套到自己 PsychoPy 收的資料上，那才會是個人作品的程度。
